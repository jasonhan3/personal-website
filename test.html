How can we make software integration more efficient?
In any company older than thirty years, software engineers face a challenging problem: how do they integrate legacy systems with their modern ones? Legacy systems that use COBOL, for example, may have data formats that aren't compatible with relational databases commonly used today, may not natively support HTTP/HTTPS protocols, and often follow a monolithic architecture rather than the microservice-based architecture prevalent in our Agile environment today. The most common solution has been to write 'scaffolding' software that bridges the legacy software to be compatible with our modern software. However, this software can be very difficult to write because of the differences in architecture. Another less common solution is to attempt to rewrite the legacy software as modern software: however, the return-on-investment of this approach is very low, as the software effectively performs the same function, and rewriting an entire system is very error-prone.
